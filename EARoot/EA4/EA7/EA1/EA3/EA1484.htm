There is an attribute "validityStatus" in Validity, which is mandatory and therefore must be defined when a validity is defined for a traffic regulation order instance. However, this attribute is not sufficient for specifying a status of a traffic regulation order. Thus the value of the "validityStatus" attribute shall be as described in the following: If the value of the "status" attribute of a traffic regulation order instance is "madeAndImplemented", "madeButNotImplemented" or "madeAndPartiallyImplemented" use "active" as the value of the "validityStatus" attribute. If the "status" is "planned" use "planned" as the value of "validityStatus". If the "status" is "partiallyWithdrawn" or "withdrawn" use "suspended".<br/>
